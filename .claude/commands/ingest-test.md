# Ingest-test: анализ одного файла без записи в граф

Цель этой команды — **сравнить уровень анализа** одного и того же источника разными моделями Claude (Opus, Sonnet, Haiku). Команда выполняет полный ingest-анализ файла, но результат пишет в отдельную папку `vault/wiki/_ingest-tests/` с именем по модели, **не трогая** реальную wiki.

## Аргумент

`$ARGUMENTS` — путь к одному файлу в `vault/raw/` (pdf / md / mp3 / mp4 / wav / m4a / webm).

Если пользователь не указал файл — покажи список доступных в `vault/raw/**` и спроси какой брать. Жди ответа.

---

## Жёсткие запреты (отличие от `/ingest`)

**НЕ ПИШИ** в:
- `vault/wiki/sources/` — не создавай страницу-источник
- `vault/wiki/index.md` — не трогай индекс
- `vault/wiki/log.md` — не пиши лог
- `vault/wiki/conflicts.md` — не пиши конфликты
- любые тематические папки (`conditions/`, `interventions/`, `anatomy/`, `concepts/`, `nutrients/`, `recipes/`, `food-groups/`, и любые другие)
- `vault/wiki/_staging/` — это для `/ingest-auto`, не для тестов

**НЕ СПРАШИВАЙ** про таксономию — это тест, не реальный ingest. Таксономические предложения просто пиши в результирующий файл.

**НЕ УДАЛЯЙ** оригиналы в `vault/raw/` даже после успешной транскрипции.

Всё, что ты пишешь в этой команде, идёт **только** в `vault/wiki/_ingest-tests/<slug>--<model>.md`.

---

## Шаги

### 1. Определи свою модель

Посмотри в системный контекст — там указано `The exact model ID is claude-<что-то>`. Преобразуй в короткий тег для имени файла:
- `claude-opus-4-7[1m]` → `opus-4-7`
- `claude-sonnet-4-6` → `sonnet-4-6`
- `claude-haiku-4-5-20251001` → `haiku-4-5`
- иначе — используй очевидный короткий вариант

### 2. Подготовь источник к чтению

- Если `.md` — читай как есть.
- Если рядом с `.pdf` уже лежит `.md` с тем же именем — читай `.md`.
- Если только `.pdf` — прогони `tools/ingest_pdf.py "<pdf>"`, дождись `.md` рядом, читай его.
- Если `.mp3 / .mp4 / .wav / .m4a / .webm` — прогони `tools/ingest_audio.sh <file> <basename-без-расширения>`; читай созданный `.md`. **Оригинал не удаляй** (в отличие от обычного ingest).

Если рядом с аудио уже есть одноимённый `.md` транскрипт — используй его, не запускай транскрипцию заново.

### 3. Прочитай источник целиком

Для длинных эфиров (>30 мин) читай посегментно по 20-30 мин, но **весь**. Качество анализа зависит от полноты прочтения — частичное чтение исказит сравнение.

### 4. Сгенерируй slug

Транслитом, дефисы, строчные, без пробелов. Например: «Питание при панкреатите» → `pitanie-pri-pankreatite`. Берётся из названия источника или имени файла.

### 5. Прочитай текущую wiki (для секций «связанные» и «конфликты»)

- `vault/wiki/index.md` — какие страницы уже есть.
- Для каждой упомянутой в источнике темы, которая уже существует в wiki, — прочитай соответствующую страницу, чтобы понять потенциальный конфликт.

Это чтение, не запись. Не трогай файлы.

### 6. Создай папку результатов если её нет

```bash
mkdir -p vault/wiki/_ingest-tests
```

### 7. Запиши результирующий файл

Путь: `vault/wiki/_ingest-tests/<slug>--<model>.md`

Формат — строго по шаблону ниже. Язык только русский. Авторский стиль сохраняй (правило №4 из CLAUDE.md) — формулировки близко к оригиналу, частичные цитаты приветствуются, канцелярит запрещён.

```markdown
---
test-run: ingest-test
file: <путь к источнику>
model: <opus-4-7 | sonnet-4-6 | haiku-4-5 | ...>
date: <YYYY-MM-DD HH:MM>
source-type: <methodology | lesson | qna | checklist | article>
slug: <slug>
---

# Ingest-test: <человекочитаемое название>

## 0. Метаданные теста
- **Модель:** <полный model ID>
- **Источник:** `<путь>`
- **Размер:** <строк / минут / страниц>
- **Дата теста:** <YYYY-MM-DD HH:MM>

## 1. Что это за источник
Тип, для кого, о чём — 2-3 предложения.

## 2. Краткое содержание
2-4 абзаца в духе автора. Не академический пересказ — авторские формулировки и интонация. Частичные цитаты приветствуются.

## 3. Ключевые тезисы
5-10 пунктов. Каждый — в авторской формулировке, с частичной цитатой где уместно.
- Тезис 1 («...»)
- Тезис 2
- ...

## 4. Проектная страница-источник
Полный предлагаемый контент для `vault/wiki/sources/<slug>.md` — как если бы это был реальный ingest. С секциями Краткое содержание / Ключевые тезисы / Связанные страницы / Цитаты. Wiki-links `[[...]]`.

## 5. Предложения для тематических страниц
### Новые
- `NEW [[<папка>/<название>]]` — обоснование 1 строкой, какой ключевой тезис из источника должен быть там
- ...

### Обновления существующих
- `UPDATE [[<папка>/<название>]]` — что дописать в секцию «Что говорят источники»
- ...

## 6. Потенциальные конфликты с текущей wiki
Для каждого — что говорит источник, что написано в wiki, где противоречие:
- `[[conditions/X]]`: автор говорит A (цитата), в wiki написано B (со ссылкой на какой источник). Противоречие — в [...]
- ...
Если не обнаружено — напиши «не обнаружено».

## 7. Предложение по таксономии
Нужна ли новая папка в `vault/wiki/`? Переименование? Переразложение? Обоснуй 1-2 строками. Если изменений не нужно — «без изменений».

## 8. Цитаты для контента
Дословные фрагменты, которые могут пригодиться для постов/сценариев. Для видео — с тайм-кодами, для pdf — со страницами если определимо.
> «...»
> «...»

## 9. Углы для контента
Потенциальные форматы постов/сценариев/рассылок на основе этого источника:
- Угол 1 — суть идеи
- Угол 2
- ...

## 10. Самооценка уровня анализа
Честно и кратко (для сравнения моделей):
- **Что я понял хорошо:** ...
- **Где я не уверен / пропустил:** ...
- **Где я домыслил или обобщил:** ...
```

### 8. Финальный отчёт пользователю

Короткий (5-6 строк):
- Модель: <tag>
- Источник: <путь>
- Результат: `vault/wiki/_ingest-tests/<slug>--<model>.md`
- Размер анализа: <сколько разделов заполнено, длина в строках>
- Новых тематических предложений: <N>, обновлений: <M>, конфликтов: <K>
- Для сравнения — запусти ту же команду в других моделях, файлы лягут рядом: `<slug>--opus-4-7.md`, `<slug>--sonnet-4-6.md`, `<slug>--haiku-4-5.md`.

## Напоминания

- Язык только русский.
- Авторский стиль — обязательно. Тест оценивает в том числе способность модели удерживать тон автора.
- **Ничего не пишется за пределами `vault/wiki/_ingest-tests/`.**
- Если файл `<slug>--<model>.md` уже существует от прошлого теста — перезапиши без вопроса. Это тест, промежуточные результаты не ценны.
- Самооценка в разделе 10 — обязательна. Это ключевой сигнал для сравнения моделей.
