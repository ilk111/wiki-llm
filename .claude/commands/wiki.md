# Query по графу wiki — shallow mode

Пользователь спрашивает: **$ARGUMENTS**

Ответь **только на основе `vault/wiki/`**. Не используй общие знания. Если в wiki данных не хватает — честно скажи «в wiki этого нет», не додумывай.

## Процедура обхода (1 хоп)

1. **Точка входа — `vault/wiki/index.md`.** Прочитай его и выбери 2-5 страниц, наиболее релевантных вопросу. Если index не даёт однозначного ответа — сделай Glob по `vault/wiki/**/*.md` и просмотри имена файлов.

2. **Прочитай выбранные страницы.** Для каждой посмотри `sources:` во frontmatter и секцию «Что говорят источники».

3. **Один хоп по `[[wiki-links]]`.** Из прочитанных страниц — прочитай напрямую упомянутые связанные страницы (только те, что выглядят релевантными вопросу, не все подряд). Глубже не иди.

4. **Если ответ требует дословных формулировок автора** — открой соответствующие `vault/wiki/sources/<slug>.md` и возьми цитаты оттуда.

5. **Не читай `vault/raw/`.** Если в wiki нет нужной информации — скажи пользователю, что для ответа нужен deep-режим или ingest нового источника.

## Формат ответа

- Ответ по существу, в духе материалов wiki (сохраняй авторские формулировки из источников).
- **Каждый тезис — с пометкой откуда**: `([[conditions/СРК]])` или `([[sources/lesson-03]])`.
- В конце — блок **«Что прочитал»**: список страниц, по которым шёл, в порядке обхода. Это trail для проверки.
- **Не создавай** новых страниц и не редактируй существующие. Если ответ тянет на отдельную страницу — в конце спроси «Зафайлить как `vault/wiki/<папка>/<slug>.md`?» и жди решения.
- **Не пиши в `log.md`** для shallow-запросов — они слишком частые, засорят лог. Пиши только если пользователь явно попросит.

## Если вопрос слишком широкий для 1 хопа

Скажи: «Вопрос требует более глубокого обхода — запусти `/wiki-deep $ARGUMENTS`». Не пытайся компенсировать глубиной сам.
